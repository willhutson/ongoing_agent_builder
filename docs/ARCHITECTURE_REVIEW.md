## 2. Agent ‚Üí ERP Integration Points

### Artifact Protocol

**Flow**: Agent creates artifact ‚Üí SSE streams to MC frontend ‚Üí User sees live building

```
Agent._execute_tool()
  ‚Üí emit_artifact(type, data)
    ‚Üí ArtifactEvent(CREATE/UPDATE/COMPLETE)
      ‚Üí asyncio.Queue ‚Üí _sse_callback ‚Üí SSE transport
        ‚Üí MC frontend renders
```

**20 artifact types** defined in `src/protocols/artifacts.py`:
- Content: calendar, brief, document, deck, moodboard
- Video: script, storyboard, shot_list
- Data: report, table, chart
- Social: social_post, social_report, listening_report
- Operations: contract, survey
- LMS: course, assessment, learning_path
- Automation: workflow

**Validation**: `validate_artifact_data(type, data)` checks required fields and types.

### Billing Hooks

All agent executions report usage to the ERP:

```
POST /api/v1/billing/usage/report
Header: X-Service-Key: <service_key>
Body: {agent_type, tokens_used, model, duration_ms, tenant_id}
```

Tier determines model and cost:
- Economy (Haiku 3.5): $0.25/$1.25 per M tokens
- Standard (Sonnet 4): $3/$15 per M tokens
- Premium (Opus 4): $15/$75 per M tokens

### Brand Context

Agents access brand data via ERP endpoints:
- `GET /api/v1/content/brand-guidelines?client_id=X` ‚Äî Voice, tone, visual rules
- `GET /api/v1/content/templates?type=X&client_id=X` ‚Äî Document templates
- Brand context is injected via the AgentFactory's 3-tier prompt assembly

### Callback Protocol

ERP sends `callback_url` + `invocation_id` ‚Üí Agent Builder sends results via PATCH with HMAC-SHA256 signature:

```
PATCH {callback_url}
Header: X-Signature: HMAC(payload, secret)
Body: {invocation_id, status, output, token_usage, duration_ms}
```

### Agent Handoff Protocol

Inter-agent delegation (e.g., SocialListening ‚Üí Observer):

```
POST /api/v1/agents/handoff
Body: {target_agent, tool, input, callback_agent, client_id}
```

Used by:
- `social_listening.delegate_to_observer` ‚Üí triggers Observer data collection
- `training_agent` ‚Üí hands off to `lms_tutor`, `lms_content`, `lms_assessment`

## 3. Social Suite Gaps

### What the ERP Social Suite Expects vs What Agents Provide

| ERP Expectation | Agent Status | Gap |
|----------------|-------------|-----|
| **Listening Dashboard** ‚Äî real-time mention feed | SocialListeningAgent calls `/api/v1/listening/*` endpoints | Endpoints must exist on ERP side. Agent is ready. |
| **Unified Inbox** ‚Äî cross-platform message triage | CommunityAgent has `get_inbox_summary`, `categorize_messages`, `draft_response` | Agent ready. ERP needs `/api/v1/social/inbox/summary` endpoint. |
| **Publishing Calendar** ‚Äî schedule/manage posts | PublisherAgent has `get_publishing_calendar`, `schedule_post` | Agent ready. ERP needs `/api/v1/publishing/calendar` and `/posts/schedule` endpoints. |
| **Sentiment Analysis** ‚Äî trend tracking | SocialListeningAgent has `analyze_sentiment_trends` | Agent calls `/api/v1/listening/sentiment/trends`. ERP needs this endpoint. |
| **Crisis Detection** ‚Äî negative spike alerts | SocialListeningAgent has `detect_crisis` | Agent calls `/api/v1/listening/crisis/detect`. ERP needs this endpoint + alert mechanism. |
| **Competitor Benchmarking** ‚Äî social performance vs competitors | CampaignAnalyticsAgent has `benchmark_against_competitors` | Agent calls `/api/v1/analytics/social/benchmark`. ERP needs this endpoint. |
| **Optimal Posting Times** ‚Äî AI-powered time suggestions | PublisherAgent `suggest_posting_times` + ContentAgent `get_optimal_posting_times` | Both call `/api/v1/publishing/optimal-times`. ERP needs historical engagement data + this endpoint. |
| **Content Adaptation** ‚Äî repurpose for platforms | PublisherAgent `adapt_for_platforms` + ContentAgent `adapt_content_for_platform` | LLM-based ‚Äî works without ERP endpoint. |
| **Social Reports** ‚Äî SOCIAL_REPORT artifact | CampaignAnalyticsAgent `generate_social_report` | Calls `/api/v1/analytics/social/reports`. Returns SOCIAL_REPORT artifact. |
| **Listening Reports** ‚Äî LISTENING_REPORT artifact | SocialListeningAgent `generate_listening_report` | Calls `/api/v1/listening/reports`. Returns LISTENING_REPORT artifact. |
| **Real-time Platform Data** ‚Äî live from Twitter, Reddit, etc. | Observer Agent with source adapters | Currently returns **mock data**. Real API integration blocked on tenant API credentials (Twitter Bearer Token, Reddit OAuth, etc.). |
| **Social Analytics Dashboard** ‚Äî aggregate metrics | SocialAnalyticsAgent (5 real HTTP + 1 stub) | Mostly production-ready. |

### Critical Gaps

1. **Observer ‚Üí Real APIs**: Source adapters are mock-only. Real Twitter/Reddit/News integration requires:
   - Per-tenant API credential management (SourceConfig)
   - Rate limiting per platform
   - Webhook/streaming support for real-time data

2. **ERP Endpoint Availability**: The Social Suite agents call 15+ new endpoints that must exist on the ERP side:
   - `/api/v1/listening/*` (4 endpoints)
   - `/api/v1/publishing/*` (6 endpoints)
   - `/api/v1/analytics/social/*` (3 endpoints)
   - `/api/v1/agents/handoff` (1 endpoint)
   - `/api/v1/social/inbox/summary` (1 endpoint)

3. **MC Router Integration**: The ERP's `mc-router.ts` needs to be updated with:
   - New agent type: `publisher`
   - New artifact formats: `social_post`, `social_report`, `listening_report`
   - Keyword detection: "schedule post", "publish to instagram", "social report", "listening report", "brand mentions", "sentiment analysis"

4. **Social Analytics Agent**: The module-level SocialAnalyticsAgent is still a stub. The production-level one (`src/agents/social_analytics_agent.py`) has 5/6 real tools but `generate_social_report` is LLM-delegated. Consider wiring it to the ERP reporting endpoint.

### Recommendations

1. **Phase 1 (ERP)**: Implement the 15 new API endpoints on the ERP side. The agent code is ready and waiting.
2. **Phase 2 (Observer)**: Wire real API adapters starting with Reddit (free API) and News (RSS/NewsAPI), then Twitter (paid API).
3. **Phase 3 (MC Router)**: Update `mc-router.ts` with publisher agent routing and new artifact type rendering.
4. **Phase 4 (Testing)**: End-to-end test: ERP ‚Üí Agent Builder ‚Üí Observer ‚Üí Source Adapters ‚Üí Listening Report artifact ‚Üí SSE ‚Üí MC frontend.
